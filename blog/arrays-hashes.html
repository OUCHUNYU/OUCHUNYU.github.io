Arrays vs. Hashes

In our DBC learning week 4, we started our Ruby study. As one of the most dynamic programming language, certainly there a lot of things going on behind the code. As this is our first week on Ruby basic syntax and iteration. It's time to reflect on what i have learned on this topic. Whatever i want to dig, i always go to their libraries, here is what i found for Array and Hash

Arrays: Arrays are ordered, integer-indexed collections of any object.This was the first sentence when you go rubydoc. In which i think it's well informative. It's purpose is to store objects, and pretty much more than 99% of stuff in Ruby are objects. So it is an everything collection. The difference is they store objects in an integer indexed order, starts from 0, from left to right. The last object in the collection is indexed -1, from right to left, it goes -2, -3, -4 and so on. We can also access a specific object inside an array by its index. Example:


          arr = ["Ruby", "Phython", "Java", "JavaScript"]

          puts arr[0]



          > Ruby # Interpreter returns the frist element in the arr.

This gives us the freedom of accessing the objects in an array. There are many build in methods to allow us to manipulate and iterate an array with ease. Because of there features, array is one of the best way to store data. You can create a new array by directly assign a pair of [ ] to a variable. You can also use Array class to generate a new array, it comes with some cool feature in creating new array this way and some weird things as well. See code below:


          # This is an empty array.
          arr = []
           # We can store many different type of data, even some arrays, because they are all objects in Ruby.
           long_arr = [1, 'ruby', :ruby, a_method_name, ['another', 'array']]
           # This creates an empty array as well
           arr_obj = Array.new

           # Note the reason why this happens because Array.new method takes integer arugment and a block behind it. The integer tells Ruby how many times we need to execute the block. In our code we did not specify a block so it returns nil for 5 times.

           arr_nils = Array.new(5) # This creates something like [nil, nil, nil, nil, nil]

Hashes: A Hash is a dictionary-like collection of unique keys and their values. Also called associative arrays, they are similar to Arrays, but where an Array uses integers as its index, a Hash allows you to use any object type. Hashes enumerate their values in the order that the corresponding keys were inserted. Like Ruby documentation says the arrys are some sort of arrays, this probobly sounds confusing but they are kind of related. One thing noticable that in Hash library, there is a method to alter an hash to an array:


            h = { "c" => 300, "a" => 100, "d" => 400, "c" => 300  }
            h.to_a
             #=> [["c", 300], ["a", 100], ["d", 400]]

Hashes are particularly useful when we need to store or access data with unique identifiers as keys. A Hash object is constructed with a key value pairs within curly brackets like the code below, and their keys are symbols, values can be any object.


          # This is a simple hash object.
          hash_1 = { ruby: "Fun", phython: "Fun, too", java: "Awesome", javascript: "I don't know"}

          # NOTE one thing in Ruby dynamic syntax we can omit { } like:
          hash_1 = ruby: "Fun", phython: "Fun, too", java: "Awesome", javascript: "I don't know"

          # To access a hash value we can do a similar syntax as arrays.
          hash_1[:ruby] # This returns "Fun"

Most of Hash and Array iterations are done by iterator methods, which inherited from Enumerator module, there is a set of hierarchy in between many classes and modules we can have them explained in future blog posts.

Ruby's dynamic feature is one of the most important key of its popularity. With this we can define methods, create personal DSLs, build Ruby gems, implimenting frameworks in a dynamic, creative ways. Happy Coding.
