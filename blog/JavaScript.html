Mainly i’m interested in Ruby Hashes and JavaScript Objects. They visually look very much the same thing, because their syntax distinctly different from each other so they both work awesomely in their own ways.

In my option, although they look so close, Ruby Hashes work more like a its purpose, they are data structures. JavaScript Objects work just like how they are defined, objects. In JavaScript, what i have read so far, methods are considered as properties of an object and variables. So when you defined a new function for a JavaScript object you have the function defined for that object, and you can access that function to have some use for that. But in Ruby Hashes, you can’t do that, you can technically assign a function you defined to a key in a hash, but what that key is going to hold is only the value of that function, which means, if your function prints a string, that key is going to return you a string, not the function itself. More obvious, when your function takes argument and you want to assign this function to a key in a Ruby Hash, you must supply all arguments it needs, otherwise Ruby console gives you an ArgumentError. Which really tells you that Ruby hash keys only hold value of an object. In JavaScript it holds the entire function, and you can call that function or even pass argument to it.
